N.b: Pour lancer le serveur de Django je dois obligatoirement faire : python manager.py runserver
pour crée une application  propre on fait  cette commande : python manage.py startapp nom_application
N.B: chaque projet  Django  à un  clé  secrette 
l'ORM c'est  la façon  que Django crée les tabls   dans la base de  données à partir de modèles que j'ai définie dans models.py
 pip freeze > requirement.txt  :  avec cette commande on pourra  facilement mettre  les librairie de l'environnement de mon projet dans le fichier requirement.txt .

ROOT_URLCONF = 'config_recettes.urls'   gère tous  ce qui  est rotage . 
c'est le domaine  de l'application : ALLOWED_HOSTS = []

je dois  obligatoirement crée un super administrateur car il a tous les permissions sur l'Application  en utilisant la commande: python manage.py createsuperuser 
pour  intaller Django on fait :   pip install Django==version 
si on a  plusieur version  de python on doit  fair   crée l'environnement actuele avec  cette commande pour  notre  projet :python(version) -m venv venv   
Pour activer l'espace  de travail de  mon projet je  dois  faire comme ça :   .\venv\Scripts\activate
Pour  désactiver on fait : desactivate 
si en veut savoir quels sont les packages et les librairie  qu'ils sont installer sur mon machine  je tape cete  commande : pip freeze  
c'estune liste  de  librairie qui vont affiché   mais  lorsque je travaill avec  l'environnent actuel j'aurai pas cette  liste 
pip c'est le gestionnaire de  package  (librairie)  dans python .

python manage.py runserver : afficher la racine et  c'est exactement  la commande qui  permet de lancer  le serveur de  l'application Django . 

===============commands de base : ===============================================================
le fichier manage.py  c'est  exactement le point  d'entrée à l'application Django ,  il contient de scripte CLI  qui  permet  d'exécuetr toute commande cli  
python manage.py makemigrations
python manage.py migrate
python manage.py runserver
python manage.py createsuperuser
